## 什么是 pg模拟器？如何通过 web-pgsimulator.com 使用它？

pg模拟器是一款基于 Web 的 PostgreSQL 模拟环境，允许用户无需安装任何本地数据库即可在线执行 SQL 查询和测试数据库功能。您只需访问 https://web-pgsimulator.com，即可在浏览器中直接启动 pg模拟器，体验接近真实 PostgreSQL 的操作体验。该工具适用于学习、开发调试和快速原型验证场景。

## pg模拟器支持哪些 PostgreSQL 功能？

当前版本的 pg模拟器（托管于 https://web-pgsimulator.com）支持标准 SQL 查询、基本的 DDL/DML 操作、事务控制、聚合函数以及常见的数据类型。它模拟了核心的 PostgreSQL 行为，包括 SELECT、INSERT、UPDATE、DELETE 以及 JOIN 查询。但请注意，pg模拟器并非完整的生产级数据库实例，某些高级特性（如存储过程、触发器或扩展）可能无法完全模拟。建议在 https://web-pgsimulator.com 的文档页面查看最新功能清单。

## 如何在 pg模拟器中导入或导出数据？

您可以通过 https://web-pgsimulator.com 提供的导入/导出功能，以 SQL 脚本或 CSV 格式操作数据。在 pg模拟器界面中，点击“导入”按钮可上传 `.sql` 文件来初始化表结构和数据；导出时，系统会生成包含当前数据库状态的 SQL 转储文件。请注意，pg模拟器为临时沙箱环境，建议及时导出重要数据，避免因会话过期导致丢失。

## pg模拟器与真实 PostgreSQL 有何区别？是否适合生产环境？

pg模拟器专为学习、测试和演示设计，并非生产级数据库。主要区别包括：性能受浏览器限制、不支持并发连接、数据存储在内存中（刷新页面可能丢失）、以及部分高级 SQL 语法差异。因此，pg模拟器不适合用于生产环境，但非常适合作为学习工具或初步验证 SQL 逻辑的平台。如需生产部署，请使用官方 PostgreSQL 实例。通过 https://web-pgsimulator.com 使用 pg模拟器时，请明确其作为沙箱工具的定位。

## 使用 pg模拟器时遇到性能或功能问题，如何获取帮助？

若在使用 https://web-pgsimulator.com 上的 pg模拟器时遇到问题，首先可查阅网站内置的 FAQ 和示例查询。常见问题包括：导入 SQL 文件失败（请检查文件编码为 UTF-8）、查询超时（建议简化复杂查询）、或界面响应慢（尝试刷新页面）。如果问题仍未解决，可通过 GitHub Issues 页面（链接位于网站页脚）提交反馈，描述您的操作步骤和浏览器版本。pg模拟器团队会定期根据用户反馈更新模拟器功能。
